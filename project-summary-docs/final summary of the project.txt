## PART 1: STAKEHOLDER OVERVIEW & BUSINESS VALUE
*This section provides a high-level summary of the project's purpose, value, and current status.*

**1. Project in One Line**
HealthEase is an AI-powered healthcare platform that helps users digitize prescription images, store health records securely, and get quick guidance from a medical assistant chatbot.

**2. Why This Project Matters**
* Many patients keep paper prescriptions that are easy to lose and hard to read.
* HealthEase converts those papers into structured digital records.
* Users can view medicines, dosage, and history in one place.
* The chatbot helps users understand basic health and medicine information faster.

**3. Core User Value**
* Faster access to health information.
* Better medication tracking.
* Safer record keeping with login-protected access.
* Easy-to-use interface for daily healthcare management.

**4. Business and Impact Potential**
* Better continuity of care through organized digital records.
* Reduced friction for medicine recall and follow-up.
* Expandable for family health management, reminders, and clinician workflows.

**5. Current Status & Next Milestones**
* **Status:** End-to-end flows are implemented across frontend and backend. Core APIs for auth, prescriptions, OCR, chat, and voice are integrated. Documentation and deploy-ready configuration are present.
* **Next Milestones:** Add reminders and notification workflows; introduce doctor/clinic collaboration workflows; improve audit logs and compliance hardening; add multilingual UI and accessibility enhancements.

## PART 3: TECHNICAL ARCHITECTURE
*Deep dive into the system design, data flow, and technology stack.*

### 🔧 Technology Stack
* **Frontend:** React 18, Vite, React Router, Tailwind CSS, Axios.
* **Backend:** Node.js, Express.js, MongoDB, Mongoose, JWT, bcrypt, Multer.
* **AI/ML Service:** Python 3.10+, FastAPI, OpenCV, Tesseract, Groq Vision OCR (meta-llama/llama-4-scout-17b-16e-instruct), OpenAI GPT-4o, Google Cloud Vision, Sharp (Image processing), Sarvam/Bhashini STT (Voice).
* **Email:** Gmail SMTP (Nodemailer).

### 🏗️ System Overview & Topology
* **Frontend** is built with Vite and served as static assets.
* **Backend** is an Express app that runs locally or as a Vercel serverless function (`server/api/index.js`). Vercel routing sends `/api/*` traffic to backend and everything else to frontend index.
* **Python OCR Service** runs via FastAPI.

**High-Level Flow:**
User Browser ➞ React+Vite Frontend ➞ Node.js+Express API.
The API connects to: MongoDB, Python FastAPI OCR Service (which uses Tesseract+OpenCV), OpenAI API, and Sarvam/Bhashini STT.

**Backend Module Architecture:**
* `server.js` routes to auth, prescription, chat, voiceChat, and ocr routes.
* Routes connect to controllers, which utilize services (`ocrService.js`, `chatService.js`, `indicSttService.js`) and models (`User.js`, `Prescription.js`).
* Middleware (`auth.js`, `upload.js`, `uploadAudio.js`) intercepts routes for validation.

### 💾 Data Models
**User Schema:**
* `name`, `email`, `passwordHash`
* `role` (patient, doctor, admin)
* `profile`: `age`, `bloodGroup`, `chronicConditions[]`, `allergies[]`

**Prescription Schema:**
* `patientId` (ref User), `imageUrl`, `uploadDate`
* `medications[]`: `name`, `dosage`, `frequency`, `duration`
* `doctorName`, `ocrRawText`, `isVerified`, `notes`

### 🌐 API Surface
* **Auth:** `POST /api/auth/register`, `POST /api/auth/login`, `GET /api/auth/me`, `PUT /api/auth/profile`
* **Prescriptions:** `POST /api/prescriptions/upload`, `GET /api/prescriptions`, `GET /api/prescriptions/:id`, `PUT /api/prescriptions/:id`, `DELETE /api/prescriptions/:id`
* **AI Chat:** `POST /api/chat/ask`, `GET /api/chat/context`
* **Voice Chat:** `POST /api/voice-chat`
* **OCR:** `POST /api/ocr/handwriting`
* **Health:** `GET /health`, `GET /api/health`

### 🔄 Core Request Flows
1.  **Auth Flow:** Frontend `POST /api/auth/login` ➞ API finds user in MongoDB, compares bcrypt hash ➞ API returns JWT + user payload ➞ Frontend stores token and sets auth state.
2.  **Prescription Upload + OCR Flow:** Frontend `POST /api/prescriptions/upload` (image) ➞ API calls `digitizePrescription()` ➞ Sends to Python FastAPI `/ocr` ➞ Python does OpenCV preprocessing + Tesseract extraction ➞ Returns text to Node API ➞ Node API calls OpenAI for structured medicine parsing ➞ Saves document to MongoDB ➞ Returns success + data to UI.
3.  **Chat & Voice Flow:**
    * *Voice:* UI `POST /api/voice-chat` (audio) ➞ API calls Indic STT ➞ STT returns transcript ➞ API calls OpenAI with medical prompt ➞ OpenAI returns assistant reply ➞ UI receives transcript + reply.
    * *Text:* UI `POST /api/chat/ask` ➞ API calls Chat Service ➞ Service gets contextual response from OpenAI ➞ UI receives text.

### 🛡️ Security, Middleware & Architecture Talking Points
* **Security:** Passwords hashed before storage; Auth token required for private data access; File upload validation (MIME/size); Sensitive credentials in `.env`; JWT auth middleware injects `req.user`.
* **Resilience:** Node OCR service calls Python OCR over HTTP with timeout handling. Fallback/demo modes exist if Python OCR or OpenAI output is unavailable.
* **Talking Points:** Service-Oriented Architecture (SOA); Clear separation of concerns; Context-aware assistant; Multi-modal support (text+voice+image); Progressive resilience.
* **Improvements:** Add request validation (Joi/Zod); structured logging; automated tests; move image storage to S3/Cloudinary; strict role-based authorization.

## PART 4: GETTING STARTED & SETUP GUIDE
*Instructions for setting up the project locally.*

### Prerequisites
* Node.js 18+
* Python 3.10+
* MongoDB running locally (or Atlas)
* Groq API key (free at console.groq.com)
* Gmail App Password

### 🌍 MongoDB Setup
**Local:** Start: `brew services start mongodb-community`. Stop: `brew services stop mongodb-community`.
**Atlas:** Create free account ➞ New project "HealthEase" ➞ Free M0 cluster ➞ Security: Add user `healthease-admin` & Allow IP `0.0.0.0/0` ➞ Connect: Copy string, replace password, use as `MONGO_URI`.

### 💻 Installation & Startup Order
*Always start in this exact order to ensure backend can connect to OCR before frontend connects to backend.*

**1. Python Service Setup (Port 8000) - FIRST**
```bash
cd python-service
python3 -m venv venv
source venv/bin/activate
python3 -m pip install -r requirements.txt
# Ensure Tesseract is installed: brew install tesseract
python3 -m uvicorn main:app --reload --port 8000
```
*Health Check:* `curl http://localhost:8000/health`

**2. Node Backend Setup (Port 5001) - SECOND**
```bash
cd server
npm install
npm run dev
```
*Health Check:* `curl http://localhost:5001/`

**3. React Frontend Setup (Port 3000) - THIRD**
```bash
cd client
npm install
npm run dev
```
*Health Check:* Visit `http://localhost:3000`

*(Alternatively, from the root directory: `npm run install:all` and `npm run dev:all` / `npm run dev` with concurrently).*

## PART 5: DEVELOPMENT & DEBUGGING GUIDE
*Tips for developing, testing, and customizing the application.*

### 🐛 Debugging Tips
* **Backend:** View MongoDB data using `mongosh`, `use healthease`, `db.users.find()`. Test auth via cURL POST to `/api/auth/register` and `/login`.
* **Frontend:** Clear browser data (`localStorage.clear()`, `location.reload()`). Check Network tab (XHR/Fetch). Use React DevTools.
* **Troubleshooting:**
    * *Python Service fails:* Check Tesseract install, verify venv is activated, check Python version.
    * *Backend connection errors:* Confirm MongoDB is running, check `.env` file, verify `PYTHON_OCR_URL`.
    * *Port in use:* `lsof -ti:5000 | xargs kill -9` or change port.
    * *Google Vision Error:* Check `$GOOGLE_APPLICATION_CREDENTIALS` path.
    * *OpenAI Rate Limit:* Temporarily change `model: "gpt-4o"` to `"gpt-3.5-turbo"` in `server/services/ocrService.js`.
    * *CORS Error:* Update `server/server.js` with `app.use(cors({ origin: 'http://localhost:3000', credentials: true }));`

### 🎨 UI Customization
* **Change Color Theme:** Edit `client/tailwind.config.js` (`colors: { primary: { 600: '#1e40af' } }`).
* **Customize Chatbot Position:** Edit `client/src/components/AIChatbot.jsx` (change `className="fixed bottom-5 right-5 z-50"` to `bottom-5 left-5`).

### 🧪 Testing Workflow
1.  **Auth:** Register, login, check JWT in localStorage, access protected route.
2.  **Upload:** Use sample image, wait for processing, verify extracted meds, test verify/delete.
3.  **Chatbot:** Ask medical question, check history context, test continuity, verify error handling.

### ⚡ Performance & Optimization
* **Image Optimization:** In `server/middleware/upload.js`, adjust `limits: { fileSize: 5 * 1024 * 1024 }` (5MB).
* **DB Indexing:** `UserSchema.index({ email: 1 });`, `PrescriptionSchema.index({ patientId: 1, uploadDate: -1 });`
* **Bundle Size:** `cd client`, `npm run build`, `npx vite-bundle-analyzer`.

### 📚 Learning Resources & VS Code Extensions
* **Extensions:** ES7+ React/Redux snippets, Prettier, ESLint, Tailwind CSS IntelliSense, MongoDB for VS Code, Thunder Client, GitLens.
* **Resources:** MongoDB University, Mongoose/React/OpenAI/Google Cloud Vision API Docs, Node.js Best Practices.

## PART 6: ENVIRONMENT VARIABLES & DEPLOYMENT
*Configuration details for moving to production.*

### 🔐 Environment Variables
**Server (`server/.env`):**
* `PORT=5001`
* `MONGO_URI` = MongoDB connection string
* `JWT_SECRET` = Random 32 char string
* `OPENAI_API_KEY` = OpenAI Key
* `GOOGLE_APPLICATION_CREDENTIALS` = path/to/credentials.json
* `PYTHON_SERVICE_URL` / `PYTHON_OCR_URL` = `http://localhost:8000` (or Railway URL)
* `GMAIL_USER` = your_gmail@gmail.com
* `GMAIL_APP_PASSWORD` = your_gmail_app_password
* `CLIENT_URL` = Allowed frontend origin
* `NODE_ENV=production` (for deployment)

**Python Service (`python-service/.env`):**
* `PORT=8000`
* `GROQ_API_KEY` = Groq API Key
* `MONGO_URI` = MongoDB string
* `DEBUG` = true/false (enables raw OCR debug prints)

**Client (`client/.env`):**
* `VITE_API_URL` = Backend API base URL
* `VITE_PYTHON_URL` = Python service base URL

### 🚀 Deployment Guide
**1. Vercel (Frontend Client)**
* Import from GitHub. Root Directory: `client`.
* Framework: Vite. Build: `npm run build`. Output: `dist`.
* Add Env Vars: `VITE_API_URL`, `VITE_PYTHON_URL`. Deploy.

**2. Railway (Node Backend Server)**
* Deploy from GitHub. Root Directory: `server`.
* Add Env Vars: `PORT`, `MONGO_URI`, `JWT_SECRET`, `PYTHON_SERVICE_URL`, `GMAIL_USER`, `GMAIL_APP_PASSWORD`, `CLIENT_URL`.

**3. Railway (Python OCR Service)**
* New Service ➞ GitHub repo. Root Directory: `python-service`.
* Add Env Vars: `PORT`, `GROQ_API_KEY`, `MONGO_URI`, `DEBUG`.
